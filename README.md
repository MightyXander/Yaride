# Yaride MVP (Telegram carpool bot)

MVP карпулинг-бота для Ярославля и ближайших населённых пунктов.

## Что реализовано

- Регистрация пользователя и выбор роли (`водитель` / `пассажир`).
- Поиск/просмотр поездок доступен **обеим ролям**.
- Создание поездок доступно только водителю.
- Бронирование доступно только пассажиру.
- Пассажир может отменить свою бронь с причиной:
  - место автоматически освобождается;
  - водителю уходит уведомление с причиной отмены.
- Полноценный выбор даты через `aiogram-calendar` (месяцы/годы/дни).
- Поиск поездок: маршрут + дата (без обязательного выбора времени).
- Смена роли доступна, если у водителя нет активных поездок на выбранную дату.
- Справочник точек маршрута:
  - популярные остановки Ярославля (по 1-2 для каждого района);
  - близлежащие города/сёла без остановок.

## Быстрый старт

1. Установить зависимости:
   - `pip install -r requirements.txt`
2. Создать `.env` на основе `.env.example`:
   - `BOT_TOKEN=...`
   - `DB_PATH=yaride.db`
3. Запустить:
   - `python main.py`

## Основные кнопки

- `Найти поездки` / `Смотреть поездки` — подбор поездок по отправлению, прибытию и дате.
- `Создать поездку` — только для водителя.
- `Мои брони` — список броней пассажира, кнопки отмены активных броней.
- `Мои поездки` — созданные поездки водителя.
- `Точки маршрута` — список остановок и близлежащих городов/сел.

## Архитектура и читаемость

Проект организован по слоям, чтобы каждый модуль отвечал за одну зону ответственности:

- `app/bot.py` — обработчики Telegram-сценариев и UI.
- `app/repo.py` — слой доступа к данным и бизнес-правила.
- `app/db.py` — подключение, транзакции и инициализация схемы SQLite.
- `app/ui.py` — фабрика клавиатур (`KeyboardFactory`) для всех UI-кнопок.
- `app/chat_ui.py` — сервис чата (`ChatUiService`) для очистки, перерисовки и истории сообщений.
- `app/trip_flow.py` — оркестратор сценариев (`TripFlowOrchestrator`) для общих переходов поиска/создания.
- `app/navigation_flow.py` — централизованная логика навигации “Назад” (`NavigationFlow`) для callback и reply-кнопок.
- `app/formatting.py` — форматирование дат/времени и строк поездок.
- `app/seeds.py` — справочник маршрутов для первичного наполнения БД.
- `app/config.py` — загрузка и валидация настроек из окружения.

В `app/repo.py` применён ООП-подход:

- Фасад `Repo` сохраняет единый API для обработчиков.
- Специализированные классы (`UserRepository`, `RouteRepository`, `TripRepository`, `BookingRepository`) разделяют ответственность по доменам.
- Общие служебные операции вынесены в базовый класс `_BaseRepository`.
- UI-составляющие также разделены по объектам: `KeyboardFactory` отвечает за клавиатуры, `ChatUiService` — за жизненный цикл сообщений.
- В `app/bot.py` сценарные переходы поиска/создания централизованы в `TripFlowOrchestrator`, что уменьшает дублирование хендлеров.
- Для ключевой логики оркестратора добавлены unit-тесты в `tests/test_trip_flow.py` (без запуска Telegram).
- Для навигации назад добавлены unit-тесты в `tests/test_navigation_flow.py`.

Такой подход уменьшает связность, упрощает чтение кода и безопасное внесение изменений.

## Ограничения MVP

- БД SQLite.
- Без геолокации и GPS, только по заранее заданным точкам.
- Без платежей (цена — только как компенсация расходов).
